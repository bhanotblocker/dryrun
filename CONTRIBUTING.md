# Contributing to Dry Run

Thanks for your interest. Dry Run is an open project shipped as a Docker image; the core implementation is maintained privately by the author. Here's how you can help.

## What we welcome

- **Bug reports.** Open an issue with reproduction steps, your OS, Docker version, and the full error output. The more specific, the faster it gets fixed.
- **Feature requests and roadmap discussion.** Comment on open roadmap issues or open new ones. Real use cases are the single most useful input.
- **Docs and visuals.** PRs to `README.md`, `CHANGELOG.md`, `docs/`, and `examples/` are accepted and reviewed. Hero images, architecture diagrams, demo GIFs, and example bundles are all great contributions.

## What we can't accept right now

- **PRs against the core engine or CLI.** These files are not part of this repository; code changes happen in a private repo and ship through the Docker image. If you'd like to collaborate on the core, reach out directly: varun.bhanot11@gmail.com.

## Reporting security issues

Please do not open public issues for security reports. Email varun.bhanot11@gmail.com directly.

## Code of conduct

Be kind. Assume good faith. We're all trying to make data engineering less painful.
